# 비동기의 본질

## 1. 동기(Sync) 방식의 흐름
- 작동: 메인 스레드가 네트워크 함수를 호출하면, 데이터가 올 때까지 그 함수 안에서 멈춰 기다림

- 문제: 호출한 함수가 끝나기 전까지는 메인 스레드는 다음 행동(코드)를 실행하지 못함. [Thread.md](./Thread.md)에서 배웠던 화면 멈춤의 원인

## 2. 비동기(Async) 방식의 흐름
- 작동: 메인 스레드가 작업을 지시하고, **즉시 제어권을 반환받아** 다음 행동(코드)로 넘어감

- 핵심: 작업은 OS나 다른 스레드가 뒤에서 따로 처리하고, 메인 스레드는 계속해서 사용자의 터치 피드백을 받거나 애니메이션을 그림
