# 앱과 서버의 통신: HTTP API

## 1. HTTP란?
비동기로 데이터를 요청하려면 목적자와 규칙이 필요하다. 스마트폰 앱이 서버와 데이터를 주고받기 위한 전 세계 공통의 약속을 HTTP라고 한다.

- **`요청 (Requset)`**: 앱이 서버에게 "데이터 좀 줘" 또는 "데이터 저장 해줘" 라고 메시지를 보낸다.
- **`응답 (Response)`**: 서버가 요청을 처리한 뒤, 결과를 앱에 돌려준다. 이때 [Asynchronous.md](./Asynchronous.md)에서 배웠던 Future가 결과값으로 채워진다.

## 2. HTTP 메서드
단순히 서버 주소만 호출하는 것이 아니라, 서버에게 '어떤 목적'으로 통신하는지 행동의 종류를 명시해야 한다. 이를 DB의 기본 작업인 CRUD(Create, Read, Update, Delete)와 매칭하여 사용한다.
- **`GET (조회)`**: 서버에서 데이터를 **가져올 때** 사용한다.
- **`POST (생성)`**: 서버에서 새로운 데이터를 **저장할 때** 사용한다.
- **`PUT/ PATCH (수정)`**: 서버에 있는 기존 데이터를 **고칠 때** 사용한다.
- **`DELETE (삭제)`**: 서버에 있는 데이터를 **지울 때** 사용한다.

## 3. JSON (데이터 규격)
서버가 앱에게 데이터를 줄 때 사용하는 가볍고 통일된 텍스트 양식이다. 사람이 읽기도 편하고 컴퓨터가 해석하기도 빠르다.
- 형태: **`{"키" : "값"}`** 쌍으로 이루어져있다.
- 예시: `{"name" : "종현", "age" : 20, "isStudent" : true}`

## 확인 퀴즈
- 상황: 기획 중인 '습관 교정 앱'에서 사용자가 '새로운 습관(매일 아침 물 마시기)'을 텍스트로 입력하고 [저장] 버튼을 눌렀다. 데이터베이스(서버)에 안전하게 저장되었다는 '완료 응답'을 받을 때까지 화면에는 '빙글빙글 도는 로딩 스피너'를 띄워야 한다.

- 질문: 이때 앱이 서버로 요청을 보낼 때 사용하는 HTTP 메서드는 4가지 중 무엇인가? 또한, 이 통신 과정을 async / await 키워드와 연관 지어 '버튼 클릭 -> 대기 -> 완료'의 흐름을 메인 스레드의 동작 관점에서 서술하시오.

내 답: POST 메서드를 사용한다. 버튼이 눌리면 `async` 함수가 실행되고, 메인 스레드는 백그라운드 스레드에게 POST 요청을 지시한 후, Future를 받는다. 네트워크 통신 함수 앞에 `await`이 붙으므로, 메인 스레드는 상자에 결과값이 찰 때까지 멈춰있는다. 그 상태에서 메인 스레드는 놀지 않고 화면에 '로딩 스피너'를 그려준다. 서버 통신이 끝나 Future에 결과값이 담기면 성공/실패에 따른 UI를 메인 스레드에서 그린다.
